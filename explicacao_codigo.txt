EXPLICACAO DO CODIGO - GERENCIAMENTO DE ESTOQUE

1) Visao geral da arquitetura
- O projeto segue separacao por responsabilidades:
  - src/App.vue: layout principal e area lateral.
  - src/router/index.ts: definicao das rotas.
  - src/views: paginas principais (Home, Admin, Usuario).
  - src/components: blocos reutilizaveis da interface.
  - src/stores/useInventoryStore.ts: estado global e regras de negocio.

2) Arquivo src/stores/useInventoryStore.ts (parte mais importante)
- Tipos principais:
  - User: id, nome, email, papel (admin/customer).
  - Product: id, nome, preco, estoque.
  - Transaction: id, tipo, produto, usuario, quantidade, total e data.
- Estado reativo:
  - users, products, transactions, currentUserId.
- Computed:
  - currentUser, isAdmin, isCustomer.
  - recentTransactions, userTransactions.
  - totalUsers, totalProducts, totalInventoryValue.
- Funcoes de negocio:
  - selectUser(id): troca usuario atual.
  - saveUser(...): cria/atualiza usuario.
  - deleteUser(id): remove usuario e ajusta usuario atual se necessario.
  - saveProduct(...): cria/atualiza produto.
  - deleteProduct(id): remove produto e transacoes relacionadas.
  - executeTransaction(productId, quantity): valida compra/venda, baixa estoque e registra transacao.
- Persistencia:
  - loadStorage e saveStorage fazem leitura/escrita no localStorage.
  - persistAll salva estado apos cada alteracao.

3) Arquivo src/router/index.ts
- Define navegacao entre:
  - / (Home)
  - /admin (painel administrativo)
  - /usuario (area do cliente)
- Usa createWebHistory para rotas amigaveis no navegador.

4) Arquivo src/App.vue
- Estrutura principal da aplicacao.
- Contem menu de navegacao e painel lateral.
- Usa componentes de layout:
  - CurrentUserCard: selecao de usuario atual e exibicao de dados.
  - QuickSummaryCard: resumo rapido de usuarios, produtos e valor em estoque.
- Renderiza as paginas com router-view.

5) Views principais

5.1) src/views/HomeView.vue
- Mostra contextualizacao inicial do sistema.
- Exibe metricas com StatCard:
  - total de usuarios
  - total de produtos
  - valor total em estoque
- Mostra ultimas transacoes com TransactionTable.

5.2) src/views/AdminView.vue
- Verifica permissao com isAdmin.
- Se nao for admin, mostra alerta de acesso restrito.
- Se for admin, permite:
  - CRUD de usuarios (UserFormCard + UserTableCard)
  - CRUD de produtos (ProductFormCard + ProductTableCard)
- Mantem estado local de edicao para saber se e cadastro novo ou atualizacao.

5.3) src/views/UserView.vue
- Verifica permissao com isCustomer e usuario atual.
- Se permitido, exibe:
  - formulario de compra (PurchaseFormCard)
  - historico de compras do usuario (TransactionTable em modo user)
- Usa executeTransaction para registrar compra com validacoes.
- Exibe feedback de sucesso/erro apos tentativa de compra.

6) Componentes reutilizaveis criados

6.1) src/components/common
- PageHeaderCard.vue: titulo + subtitulo padrao de secao.
- StatCard.vue: card de metrica simples.
- TransactionTable.vue: tabela reutilizavel para dois contextos:
  - mode='recent' (Home)
  - mode='user' (Area do cliente)

6.2) src/components/admin
- UserFormCard.vue: formulario de cadastro/edicao de usuarios.
- UserTableCard.vue: listagem de usuarios com acoes editar/excluir.
- ProductFormCard.vue: formulario de cadastro/edicao de produtos.
- ProductTableCard.vue: listagem de produtos com acoes editar/excluir.

6.3) src/components/user
- PurchaseFormCard.vue: formulario para selecionar produto, quantidade e confirmar compra.

6.4) src/components/layout
- CurrentUserCard.vue: controla selecao de usuario atual no layout.
- QuickSummaryCard.vue: exibe resumo numerico lateral.

7) Fluxo de dados (resumo)
- Usuario interage com formulario/tela.
- View chama funcao do store.
- Store valida regra de negocio.
- Store atualiza estado reativo.
- Store persiste no localStorage.
- Interface atualiza automaticamente via reatividade do Vue.

8) Beneficios da organizacao atual
- Menos duplicacao de codigo.
- Maior clareza das responsabilidades.
- Facilidade para testes e manutencao.
- Escalabilidade para adicionar novas telas/recursos.

9) Melhorias futuras sugeridas
